# Goal
03_outline をもとに、
読者の「理解」「納得」「行動」を導く卓越した見出し構成を生成する。

# Quality Rules
・見出しだけで要点が理解できる（抽象禁止）
・誇張・煽り表現禁止（例：最強、革命、神）
・英語固有名は正確に記述
・数値は「出典あり」のみ採用（不明は”不明”と明記）
・時系列/因果関係が明確（MECE）
・無料/有料の境目を明示
・判断根拠をコメントで付与（AIの思い込み排除）

# Dependency Rules
・03_AO の outline と 1:1 で対応する見出しを順序通りに作成する（追加・削除禁止）
・`paid_start_id` は有料パートで最初に来る見出しの id を指定する

# Output Format (JSON)
{
  "headings": [
    {
      "id": "F1",
      "type": "H2",
      "text": "例：Supademoが解決した課題",
      "why": "その見出しを採用した理由（論理）"
    }
  ],
  "paid_start_id": "P1" //ここからが有料
}
